These are the webpages that list descriptions for each component, sorted by project. Each component will have the following sections:

Component Name

The following options are not all compulsory.

Information

creator:

Organisation or individual owner of component.

Subject:

Project name.

Source:

Location of source file within project.

Publisher:

Publisher of component.

Rights:

Specific licence for the component.

Description:

Description of the component.

Implements

Reference:

The component is based on this published material.

Equation:

Governing equation for the component.

Code

Inherits:

The parent the component inherits.

Children:

Children of the component.

Example Code:

XML example of code use.

Parameters:

Table of parameters that component can define.

Associations:

Components and values that this component requires to be defined.

Equation:

Image of equation governing the component.
