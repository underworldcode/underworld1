These are the webpages that list descriptions for each component, sorted by project. Each component will have the following sections:

Component Name

The following options are not all compulsory.

Organisation:

Organisational owner of component.

Project:

Project name.

Location:

Location of src file within project.

Project Web:

Project URL

Copyright:

Copyright holder.

License:

Specific licence of the component.

Parent:

The component's parent component

Children:

Children of the component.

Description:

Description of the component.

Example:

XML example of code use.

Equation:

Image of equation governing the component.

Params:

Table of parameters that component can define.

Dependencies:

Components and values that this component requires to be defined.
